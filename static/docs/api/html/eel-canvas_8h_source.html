73 /* EelCanvasItem - base item class for canvas items
74 *
75 * All canvas items are derived from EelCanvasItem. The only information a
76 * EelCanvasItem contains is its parent canvas, its parent canvas item group,
77 * and its bounding box in world coordinates.
78 *
79 * Items inside a canvas are organized in a tree of EelCanvasItemGroup nodes
80 * and EelCanvasItem leaves. Each canvas has a single root group, which can
81 * be obtained with the eel_canvas_get_root() function.
82 *
83 * The abstract EelCanvasItem class does not have any configurable or
84 * queryable attributes.
85 */
86

294 /* EelCanvasGroup - a group of canvas items
295 *
296 * A group is a node in the hierarchical tree of groups/items inside a canvas.
297 * Groups serve to give a logical structure to the items.
298 *
299 * Consider a circuit editor application that uses the canvas for its schematic
300 * display. Hierarchically, there would be canvas groups that contain all the
301 * components needed for an "adder", for example -- this includes some logic
302 * gates as well as wires. You can move stuff around in a convenient way by
303 * doing a eel_canvas_item_move() of the hierarchical groups -- to move an
304 * adder, simply move the group that represents the adder.
305 *
306 * The following arguments are available:
307 *
308 * name type read/write description
309 * --------------------------------------------------------------------------------
310 * x double RW X coordinate of group's origin
311 * y double RW Y coordinate of group's origin
312 */
313
