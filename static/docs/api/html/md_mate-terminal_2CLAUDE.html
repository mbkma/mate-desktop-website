mate-terminal GtkApplication Refactoring - COMPLETED

Summary

mate-terminal has been refactored from a legacy gtk_main() + GObject singleton pattern to a proper GtkApplication-based architecture, preparing it for future libmateui adoption.

The menu system has been fully migrated from deprecated GtkUIManager to modern GtkBuilder/GMenu.

Architecture After Refactoring

Application Structure

  • TerminalApp - Now inherits from GtkApplication (was GObject singleton)
  • TerminalWindow - Now inherits from GtkApplicationWindow (was GtkWindow)
  • Application ID: org.mate.Terminal
  • Single-instance: Handled by GtkApplication (replaced custom D-Bus factory)

Key Files Modified

  • src/terminal.c - Uses g_application_run() (was gtk_main())
  • src/terminal-app.c/h - GtkApplication subclass with startup/activate/command_line/shutdown vfuncs
  • src/terminal-window.c/h - GtkApplicationWindow subclass with GAction/GMenu support
  • src/terminal-accels.c/h - Added GAction accelerator support via gtk_application_set_accels_for_action()

New Files Created

  • src/terminal-actions.c/h - Centralized GAction definitions for app and window
  • src/terminal-menus.ui - GMenu-based menu definitions (main menubar, popup, notebook popup)

Files Removed

  • src/terminal-tabs-menu.c/h - Replaced by GMenu-based tabs menu in terminal-window.c

GAction System

Application-level Actions (prefix: app.)

  • quit - Quit application
  • new-window - Open new terminal window
  • preferences - Show profile manager
  • help - Show help

Window-level Actions (prefix: win.)

  • new-tab, close-tab, close-window
  • copy, paste, select-all
  • zoom-in, zoom-out, zoom-normal
  • fullscreen, menubar
  • find, find-next, find-prev
  • set-title, reset, reset-clear
  • prev-tab, next-tab, move-tab-left, move-tab-right, detach-tab
  • new-profile, keybindings, current-profile
  • about

Accelerator System

  • GAction accelerators: Set via gtk_application_set_accels_for_action() - these handle actual key events
  • Legacy accel_map: Still used by GtkBuilder menus for display and keybinding editor
  • GSettings: Source of truth for both systems
  • Synchronization: When GSettings changes, both systems are updated

Menu System (GtkBuilder/GMenu)

The menu system has been fully migrated from deprecated GtkUIManager to GtkBuilder/GMenu:

Menu Structure

  • Main Menubar: Defined in terminal-menus.ui, created with gtk_menu_bar_new_from_model()
  • Context Popup: GMenu-based popup for terminal screen
  • Notebook Popup: GMenu-based popup for tab management
  • Dynamic Sections: Profiles, encodings, and tabs menus are populated dynamically

Dynamic Menu Sections

  • profiles-section - Profile switching in Terminal menu
  • popup-profiles-section - Profile switching in popup
  • new-window-profiles-section - New window with profile
  • new-tab-profiles-section - New tab with profile
  • encodings-section - Character encoding selection
  • tabs-section - Dynamic tab switching menu

Tab Switching

  • Tab menu items use win.switch-tab action with integer parameter
  • Menu is updated automatically when tabs are added/removed/reordered/renamed

What Was Removed

  • global_app static singleton variable
  • settings_global global GSettings variable (now instance field)
  • detach_tab global flag (replaced with internal function parameter)
  • Custom D-Bus factory code (GtkApplication handles single-instance)
  • Direct gtk_main() usage
  • GtkUIManager: Fully replaced by GtkBuilder/GMenu
  • GtkActionGroup: Replaced by GAction entries in terminal-actions.c
  • TerminalTabsMenu: Tabs menu now uses GMenu in terminal-window.c
  • terminal_window_get_ui_manager() function

What Remains (Intentionally)

  • GtkBuilder: Used for menus and UI definitions
  • Legacy GtkAccelGroup/accel_map: Retained for menu accelerator display and keybinding editor UI
  • EggSMClient: Session management, properly integrated with GtkApplication lifecycle
  • Some legacy GtkAction callbacks: Will be removed in future cleanup pass

GSettings Schemas (Unchanged)

  • org.mate.terminal.global - Global settings
  • org.mate.terminal.profile - Per-profile settings
  • org.mate.terminal.keybindings - Keyboard shortcuts

Build Requirements

meson setup build
meson compile -C build

Testing

./build/src/mate-terminal --version
./build/src/mate-terminal # Launch terminal

Future libmateui Integration Points

When libmateui is available, these mappings apply:

  • TerminalApp -> MateUiApplication
  • terminal-accels.c -> MateUiAccel
  • EggSMClient -> MateUiSession
  • About/message dialogs -> MateUiDialogs

The GAction infrastructure is now in place, making this transition straightforward.
