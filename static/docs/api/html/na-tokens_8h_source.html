33/* @title: NATokens
34 * @short_description: The #NATokens Class Definition
35 * @include: core/na-tokens.h
36 *
37 * The #NATokens class manages the tokens which are to be replaced with
38 * elements of the current selection at runtime.
39 *
40 * Note that until v2.30, tokens were parsed against selection list only
41 * when an item was selected in the Caja context menu (i.e. at
42 * execution time).
43 * Starting with unstable v2.99 (stable v3.0), this same parsing may occur
44 * for each displayed label (as new specs accept tokens in labels) - we so
45 * factorize this parsing one time for each new selection in the Caja
46 * plugin, attaching the result to each item in the context menu.
47 *
48 * Adding a parameter requires updating of:
49 * - doc/cact/C/figures/cact-legend.png screenshot
50 * - doc/cact/C/cact-execution.xml "Multiple execution" paragraph
51 * - src/core/na-tokens.c::is_singular_exec() function
52 * - src/core/na-tokens.c::parse_singular() function
53 * - src/cact/caja-actions-config-tool.ui:LegendDialog labels
54 * - src/core/na-object-profile-factory.c:NAFO_DATA_PARAMETERS comment
55 *
56 * Valid parameters are :
57 *
58 * %b: (first) basename
59 * %B: space-separated list of basenames
60 * %c: count of selected items
61 * %d: (first) base directory
62 * %D: space-separated list of base directory of each selected item
63 * %f: (first) file name
64 * %F: space-separated list of selected file names
65 * %h: hostname of the (first) URI
66 * %m: (first) mimetype
67 * %M: space-separated list of mimetypes
68 * %n: username of the (first) URI
69 * %o: no-op operator which forces a singular form of execution
70 * %O: no-op operator which forces a plural form of execution
71 * %p: port number of the (first) URI
72 * %s: scheme of the (first) URI
73 * %u: (first) URI
74 * %U: space-separated list of selected URIs
75 * %w: (first) basename without the extension
76 * %W: space-separated list of basenames without their extension
77 * %x: (first) extension
78 * %X: space-separated list of extensions
79 * %%: the « % » character
80 */
